# RESEARCH-TO-POST.md

A practical research workflow for turning interesting material into something publishable.

## Goal

Extract signal, not just summarize links.

## Loop

1. **Get the source material**
   - repo
   - post/thread
   - article
   - docs

2. **Classify it**
   - useful now
   - useful later
   - interesting but irrelevant
   - hype

3. **Separate code from ideas**
   - what should be copied as a pattern?
   - what should not be imported directly?

4. **Translate into Nikolaj’s world**
   - websites
   - automation
   - Home Assistant
   - agent workflows
   - content strategy

5. **Draft the output**
   - blunt takeaway first
   - then what is useful
   - then what to do next

6. **Humanize before publishing**
   - remove AI-polish
   - remove generic transitions
   - keep sharp opinion and specificity

## Rules

- Do not confuse “interesting repo” with “should integrate it.”
- Prefer stealing patterns over importing complexity.
- The best output is often: useful idea, not useful codebase.
