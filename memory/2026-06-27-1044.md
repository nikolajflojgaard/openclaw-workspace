Google has two different tokens:
• **Access token:** short-lived, usually about 1 hour.
• **Refresh token:** what `gog` needs. This usually has **no fixed short expiry** if the OAuth app is in production and the token keeps being used.

The common failure cases are:
• **7 days:** if the OAuth app is External + Testing mode.
• **6 months unused:** Google can expire a refresh token if it has not been used for six months.
• revoked manually in Google account security settings.
• password changed, especially if Gmail scopes are involved.
• too many refresh tokens issued for the same user/client.
• time-limited access was granted.

For our `gog` use, I would not assume “good for 14 days”. I’d treat it as long-lived *once authorized properly*, but fragile enough that we should have a remote recovery path.
