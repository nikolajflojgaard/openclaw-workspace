# Memory Boundaries

## Promote

Promote facts that are stable, reusable, and likely to improve future decisions:

- user preferences
- durable project context
- standing decisions
- recurring workflows
- repeated lessons
- important people or systems when relevant
- changes to how the agent should behave

## Keep Daily Only

Keep raw chronology in daily notes:

- task-by-task logs
- temporary blockers
- command output
- one-off links
- rough brainstorms
- transient status

## Do Not Store

Do not store:

- secrets or tokens
- credentials
- private data with no durable need
- unverified speculation
- emotionally loaded interpretation when a concrete fact will do

## Move Elsewhere

Some durable items do not belong in `MEMORY.md`:

- agent behavior rules: `AGENTS.md`
- local setup notes: `TOOLS.md`
- reusable workflows: a skill
- debugging procedure: `DEBUG-LOOP.md`
- research-to-writing process: `RESEARCH-TO-POST.md`
